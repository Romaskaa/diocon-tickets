# Roadmap: Тикет системы

## Сейчас (май 2026)

 - [ ] **Альфа тестирование**
    - Развёртывание приложения на тестовом сервере, в приближенной к production середе
    - Сбор обратной связи (по улучшению UI/UX, доработке функционала)
    - Покрытие большей части сценариев тестами - coverage 92% + (интеграционные + end-to-end)
    - Доработка системы уведомлений

 - [ ] **Интеграция с 1С УФФ + учёт времени и трудозатрат**
    - Настройка long-polling для обмена тикетов
    - Автоматизация процесса согласования ЛУРВ-ов клиентом (листов учёта рабочего времени)
    - Расчёт предварительного времени выполнения тикета, используя AI
    - Авто-списание часов после согласования ЛУРВ

 - [ ] **UI/UX**
    - Добавление светлой темы
    - Возможность вставлять медиа в текст
 
 - [ ] **Безопасность**
    - Ограничение запросов по IP-адресу на уровне сервера
    - Проверка паролей на надёжность + предложения по генерации надёжного пароля
    - Реализовать механизм сброса пароля

## Июнь 2026

 - [ ] **Запуск в промышленную эксплуатацию**
    - Развёртывание приложения на публичном Web-сервере
    - Настройка домена и SSL сертификата

 - [ ] **Добавление новых каналов создания заявок**
    - Email, виджет на сайте, ...

 - [ ] **Разработка базы знаний**
    - Проектирование структуры БЗ (разделение на клиентскую и внутреннюю)
    - Ведение БЗ по решениям тикета, по проекту, по контрагенту
    - Поддержка мульти-модальности: текст, изображения, HTML, ссылки (URL), MS Office, ...
    - Внедрение AI (Knowledge graph RAG)
 
 - [ ] **AI функционал**
    - Умный поиск подобных тикетов. После создания тикета алгоритм находит ТОП 5 похожих тикетов, среди которых может быть решение проблемы
    - Побор наилучших кандидатов, на которых можно назначить тикет. Требует заполнения профиля сотрудника поддержки (навыки, стек технологий, ...)

## Июль 2026

 - [ ] **Добавление линий поддержки**
    - Авто эскалация и распределение тикетов
    - Внедрение AI в 1 линию поддержки

 - [ ] **Реализация встроенного мессенджера**
    - Группы + ЛС
    - Ответы на сообщения
    - Поддержка медиа контента
    - Реакции + ращличный интерактив (опросы)
    - Политика уведомлений
    - ? Сквозное шифрование
    - Возможность создать заявку из чата

 - [ ] **SLA + метрики и дашборды**
    - Настройка SLA политик, подробные метрики
    - Kanban доска
    - Детальная аналитика:
       - Выявления "проблемных" программных продуктов (те продукты/релизы с самым большим кол-вом заявок)
       - Классификация тикетов
