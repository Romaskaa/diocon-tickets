# Ролевая модель для работы с комментариями тикета

| Роль            | Может оставлять комментарии | К каким тикетам                                 | Какие типы комментариев | Примечание                                  |
|-----------------|-----------------------------|-------------------------------------------------|-------------------------|---------------------------------------------|
| CUSTOMER        | Да                          | Только свои (reporter_id == user_id)            | Только PUBLIC + NOTE    | Клиент видит и пишет только в своих тикетах |
| CUSTOMER_ADMIN  | Да                          | Все тикеты своего контрагента (counterparty_id) | Только PUBLIC + NOTE    | Может комментировать все тикеты компании    |
| SUPPORT_AGENT   | Да                          | Все тикеты (или в рамках своих проектов)        | PUBLIC, INTERNAL, NOTE  | Полный доступ                               |
| SUPPORT_MANAGER | Да                          | Все тикеты                                      | Все типы                | Полный доступ + модерация                   |
| ADMIN           | Да                          | Все тикеты                                      | Все типы                | Полный доступ                               |

## Ключевые правила:

 - **CUSTOMER** — может комментировать только те тикеты, где он является reporter_id.
 - **CUSTOMER_ADMIN** — может комментировать все тикеты своего counterparty_id.
 - **Поддержка (SUPPORT_AGENT и выше)** — может комментировать любой тикет в системе (или в рамках разрешённых проектов).
 - Клиенты никогда не могут писать INTERNAL комментарии.

## Важные моменты

 - Сотрудники поддержки могут комментировать все тикеты (не ограничений). т.е
сотрудник поддержки может оставить комментарий к тикету на который он не назначен.

| Аргумент          | Объяснение                                                                                                                                                                |
|-------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Гибкость команды  | Агенты часто помогают друг другу. Один агент может быть в отпуске, болеть, или просто перегружен. Другой агент должен иметь возможность быстро ответить в тикете коллеги. |
| Коллаборация      | Поддержка — это командная работа. INTERNAL комментарии как раз для этого и нужны ("смотри, клиент пишет вот это...").                                                     |
| Скорость реакции  | Если агент видит тикет в очереди и может быстро ответить — это плюс для SLA.                                                                                              |
| Реальная практика | В большинстве зрелых тикет-систем (Zendesk, Jira Service Management, Freshdesk и др.) агент может комментировать любой тикет, к которому у него есть доступ.              |
