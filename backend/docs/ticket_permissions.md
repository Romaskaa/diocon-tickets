# Кто может создавать и менять статус тикетов (ролевая модель)

| Действие                        | Customer | Support Agent | Executor (Developer) | Support Manager | Admin |
|---------------------------------|----------|---------------|----------------------|-----------------|-------|
| Создать тикет                   | Yes      | Yes           | Yes                  | Yes             | Yes   |
| Изменять статус тикета          | No       | Yes           | Yes (ограниченно)    | Yes             | Yes   |
| Назначать исполнителя           | No       | Yes           | No                   | Yes             | Yes   |
| Добавлять публичный комментарий | Yes      | Yes           | Yes                  | Yes             | Yes   |
| Добавлять internal/note         | No       | Yes           | Yes                  | Yes             | Yes   |
| Редактировать свой комментарий  | Yes      | Yes           | Yes                  | Yes             | Yes   |
| Просматривать все тикеты        | No       | No            | No                   | Yes             | Yes   |

## Вывод по ролям:

 - **Customer** — может только создавать тикет и писать публичные комментарии.
 - **Support Agent** — основная рабочая роль.
 - **Executor** — человек, которому тикет назначают (разработчик, дизайнер, аналитик и т.д.).
 - **Support Manager** — может всё, что может агент + распределение тикетов.
