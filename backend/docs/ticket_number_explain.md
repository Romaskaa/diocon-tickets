# Пояснения к номеру тикета

**Номер тикета** - уникальная строка состоящая из латинских заглавных букв и цифр от 0 до 9, а также двух тире, обеспечивающаяся идентичность тикета в рамках системы.

## Формат номера

`PREFIX`-`YY`-`SEQUENCE`

 - **PREFIX** - Длина от 1 до 10 символов. Может быть ключом проекта, транслитерированным наименованием контрагента или 
 же 'INT' - если тикет внутренний.
 - **YY** - Последние 2 цифры года, когда был создан тикет (2026 -> 26).
 - **SEQUENCE** - 8 разрядная целочисленная последовательность, например: 00000001 - значит, что это первый тикет.

## Ёмкость на один префикс в год

`10^8 = 100 000 000`

## Общая ёмкость

Пусть имеется 1000 активных префиксов, то в год система способна уместить:

`1000 * 100 000 000 = 100 млрд тикетов в год`

> Очень большой запас

## Вероятность коллизии

При успешной атомарной генерации + префикс уникален, то вероятность коллизии **строго равна нулю**.

### Алгоритм создания номера

```python
@classmethod
def _create(cls, prefix: str, total_tickets: int) -> Self:
    """Создание номера тикета"""

    # 1. Проверка количества тикетов
    if total_tickets < 0:
        raise ValueError("Total tickets cannot be negative")

    # 2. Год (две последние цифры)
    year_short = current_datetime().year % 100

    # 3. Числовая последовательность, последние 8 цифр
    sequence = f"{total_tickets + 1}".zfill(cls.SEQUENCE_LENGTH)

    # 4. Формирование номера
    number = f"{prefix}-{year_short:02d}-{sequence}"
    return cls(value=number)
```

`total_tickets` - актуальное число тикетов на момент вставки новой записи.

## Применение

На клиенте номер тикета и ключ проекта должен использоваться в URL маршрутах, например:

`https://diocon.desk/my-tickets/SOGLASIE-26-00000001` - пример URL адреса, реальный адрес может быть другим.

Используется для просмотра конкретного тикета.
